# Experimental Configuration Archive

This directory contains experimental configurations that were created during the development and testing phases of the multimodal librarian project. These files have been archived as part of the configuration cleanup process to maintain a clean, canonical production setup.

## Archive Structure

- **ai-enhanced/**: AI-enhanced deployment experiments
- **enhanced-core/**: Core enhancement experiments  
- **enhanced-final/**: Final enhancement experiments
- **enhanced-minimal/**: Minimal enhancement experiments
- **full-ml-variants/**: Full ML deployment variants
- **functional-chat/**: Functional chat experiments
- **learning-deployment/**: Learning deployment configurations
- **legacy/**: Legacy configurations
- **self-contained/**: Self-contained deployment experiments
- **simple/**: Simple deployment experiments
- **temporary-patches/**: Temporary patches and fixes
- **testing/**: Testing configurations
- **websocket-experiments/**: WebSocket functionality experiments

## Purpose

These configurations were experimental attempts to:
- Test different deployment strategies
- Experiment with various feature combinations
- Validate different architectural approaches
- Provide temporary fixes during development

## Current Status

All experimental configurations have been archived. The canonical production configuration consists of:
- `main.py` - Single main application file
- `Dockerfile` - Single production Dockerfile
- `requirements.txt` - Consolidated requirements
- `scripts/deploy.sh` - Single deployment script
- `task-definition.json` - Single task definition
